# プロジェクトルール

## AI生成記法の残存防止ルール
Web公開用のHTMLファイルなどを生成・編集する際は、AIによる生成であることを想起させる記法やテキストが一切残らないように以下の項目を徹底してください。

1. **Markdown表記の混入禁止**
   * 強調を示す `**` や `*` をHTMLファイル内にテキストとして埋め込まないでください。太字にする場合は必ず `<strong>` や `<b>` タグを使用するか、CSSでスタイリングしてください。
   * リンクを示す `[テキスト](URL)` や見出しを示す `#` をHTMLファイル内にテキストとして残さないでください。
2. **AI特有のメタテキストの排除**
   * 記事やコンテンツの本文中に「以下に記事を作成します」「以上が内容になります」といった、システムプロンプトへの応答を示す前置きや後書きを絶対に含めないでください。
3. **品質チェックの義務化**
   * ファイルを生成または編集した後は、必ず `**` などのMarkdown記号がファイル内に残留していないか検索・検証を行ってください。

## コンプライアンス（法適合性）遵守ルール
Webサイトやコラム内で不動産の有料引き取り、名義変更、登記手続き等について執筆・編集する際は、宅建業法、司法書士法、弁護士法（非弁行為防止）等に適合するよう以下の項目を徹底してください。

1. **契約主体と登記代理人の役割分担の明確化**
   * 不動産の取引・引き受け契約自体の主体は「当センター（または提携する宅地建物取引業者）」であることを明記してください（無免許での宅建業行為と誤解されないようにするため）。
   * 登記申請の代理や戸籍の収集などの実務は「提携/専属司法書士が代理する」と明記し、司法書士が契約の媒介や交渉などを一括して代行しているかのような誤解を招く表現を避けてください（非弁行為や司法書士法の逸脱とみなされないようにするため）。
2. **「司法書士が一括代行する」といった曖昧な表現の禁止**
   * 契約書の締結といった取引行為と、登記などの法的な代理手続きを明確に書き分け、コンプライアンス（法適合性）に則った表現を使用してください。
