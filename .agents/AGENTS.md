# プロジェクトルール

## AI生成記法の残存防止ルール
Web公開用のHTMLファイルなどを生成・編集する際は、AIによる生成であることを想起させる記法やテキストが一切残らないように以下の項目を徹底してください。

1. **Markdown表記の混入禁止**
   * 強調を示す `**` や `*` をHTMLファイル内にテキストとして埋め込まないでください。太字にする場合は必ず `<strong>` や `<b>` タグを使用するか、CSSでスタイリングしてください。
   * リンクを示す `[テキスト](URL)` や見出しを示す `#` をHTMLファイル内にテキストとして残さないでください。
2. **AI特有のメタテキストの排除**
   * 記事やコンテンツの本文中に「以下に記事を作成します」「以上が内容になります」といった、システムプロンプトへの応答を示す前置きや後書きを絶対に含めないでください。
3. **品質チェックの義務化**
   * ファイルを生成または編集した後は、必ず `**` などのMarkdown記号がファイル内に残留していないか検索・検証を行ってください。

## コンプライアンス（法適合性）遵守ルール
Webサイトやコラム内で不動産の有料引き取り、名義変更、登記手続き等について執筆・編集する際は、宅建業法、司法書士法、弁護士法（非弁行為防止）等に適合するよう以下の項目を徹底してください。

1. **契約主体と登記代理人の役割分担の明確化**
   * 不動産の取引・引き受け契約自体の主体は「当センター（または提携する宅地建物取引業者）」であることを明記してください（無免許での宅建業行為と誤解されないようにするため）。
   * 登記申請の代理や戸籍の収集などの実務は「提携/専属司法書士が代理する」と明記し、司法書士が契約の媒介や交渉などを一括して代行しているかのような誤解を招く表現を避けてください（非弁行為や司法書士法の逸脱とみなされないようにするため）。
2. **「司法書士が一括代行する」といった曖昧な表現の禁止**
   * 契約書の締結といった取引行為と、登記などの法的な代理手続きを明確に書き分け、コンプライアンス（法適合性）に則った表現を使用してください。

## コラム作成時のペルソナ設定・活用ルール
コラム（記事）を作成・執筆する際は、以下のステップとルールを厳守してください。

1. **ペルソナの定義（前提条件）**
   コラム執筆を開始する前に、必ず以下の要素を満たす明確なターゲットペルソナを設定し、明記または設計してください。
   * **対象者（どんな人か）**: 年齢、家族構成、職業、所有する物件の種類や所在地などのプロフィール
   * **課題（何に困っているか）**: 現在抱えている具体的な問題やストレス、放置せざるを得ない背景など
   * **成功定義（どのような悩みを解決したら成功か）**: コラムを通じて読者がどのような状態になれば解決・成功と言えるのか
2. **執筆プロセスにおけるペルソナの引き合い**
   * 本文の導入部や展開部、まとめ等において、設定したペルソナの悩みや状況を繰り返し引き合いに出し、読者が「自分ごと」として捉えられるように執筆してください。
   * 提示する解決策が、設定したペルソナの課題をどう解消し、成功定義にどう導くのかを論理的かつ共感を持って説明してください。

## モバイルファースト・レスポンシブデザイン遵守ルール
Webサイト（LPやコラムなど）を開発・編集する際は、スマートフォン環境（画面幅768px以下、特に375pxなどの極小画面）で要素が横方向に見切れたり、不要な横スクロールが発生したりしないよう、以下の項目を徹底してください。

1. **余白（padding/margin）のスマホ最適化**
   * コラム本文やコンテンツカード等、全体の幅を規定するコンテナ要素では、スマホ表示時の左右余白を `1.5rem` (24px) 〜 `1rem` (16px) 以下に抑え、画面幅をはみ出さないようにしてください。
2. **ヘッダーなどの横並び要素の折り返し設計**
   * ヘッダーのサブリンク等、複数のナビゲーションボタン群を横並びにする際は、`:has()` 擬似クラスのような一部のブラウザで動作が不安定な指定に依存せず、スマホ表示時は `flex-wrap: wrap` や `flex-direction: column` を用いて自動的に折り返して2段構成になるように実装してください。
3. **表（テーブル）のレスポンシブ化**
   * 横長で情報量が多い表を掲載する場合は、表自体の幅を固定（`min-width: 900px` 等）しつつ、必ず `overflow-x: auto;` と `-webkit-overflow-scrolling: touch;` を指定したレスポンシブ用のラッパーコンテナ（例: `.table-responsive`）で包んでください。これにより、親要素を突き破って全体のレイアウトが横に崩れるのを防ぎ、テーブルエリア内のみで横スクロールできるようにします。
4. **テキストやタイトルの折り返しとサイズ調整**
   * コラムタイトルなどの大きな見出しは、スマホ表示時に適切なフォントサイズ（`1.3rem` 〜 `1.5rem` 程度）に縮小されるようにメディアクエリで調整してください。
   * パンくずリストや長い単語等、折り返されずに画面外へはみ出やすい要素には、適宜 `word-break: break-all` または `overflow-wrap: break-word` を設定し、自動的に改行されるようにしてください。


## 負動産（個人所有の墓地・空き家・山林等）処分コンテンツ 執筆エージェントルール

### 目的
「地目が墓地の土地」や「空き家」「山林」等の負動産の処分に関する記事は、読者の不安につけ込んで自社の有料サービスへ誘導する構成になりやすい傾向があります。本ルールは、事実として誤りはなくても「読者が他に選択肢がないと誤解する」構成・語彙を防ぎ、信頼性を保ちながら自社サービスも適切に紹介できる状態を維持するためのものです。

---

### 1. 禁止表現（NGワード・言い回し）

以下のような、断定・誇張・不安煽動につながる表現は使用しない。

| NG | 理由 | 代替表現の例 |
|---|---|---|
| 「100%不可能」「絶対に〜できない」 | 正規の手続きや状況次第で覆る可能性を隠す | 「原則として困難」「〜のままでは難しい」 |
| 「絶望」「もう手遅れ」等の感情煽動語 | 冷静な判断を妨げる不安喚起 | 「お困りの方が多い状況」 |
| 「超特殊な負動産」等のレッテル的強調表現 | 過度なドラマ化 | 「処分に手間がかかる土地」「管理が困難な不動産」 |
| 「〜しかありません」「唯一の方法」 | 代替ルートの存在を消してしまう | 「選択肢の一つとして」 |
| 検証していない具体的な法令・要件名の断定 | 誤情報のリスク | 出典を確認できた範囲のみ記載、不明な場合は「一般的には」と留保 |
| 「1円でも売れない」「1円でも〜ない」 | 売買成立の可能性が完全ゼロであるかのように言い切る | 「原則として売却が困難」「〜のケースが多い」 |
| 「確実に解決できます」「最も確実な」 | 契約や解決を100%保証するような表現 | 「解決できるケースが多い」「解決の選択肢がある」 |
| 「最終手段」 | 他の解決策（正規ルート）を排除し、自社サービスしかないと誤解させる | 「解決のための選択肢」「手放し方法の一つ」 |
| 「今すぐ止めませんか」等の緊急・煽動表現 | 焦燥感を煽り冷静な判断を妨げる | 「見直してみませんか」「解消に向けて相談してみませんか」 |

**判定基準**：見出し・リード文だけを読んだ読者が「自社サービス以外に道はない」と誤解する書き方になっていないか、を必ず自問する。

---

### 2. 情報の提示順序（構成ルール）

不安を煽ってから救済策（自社サービス）を出す構成を避け、必ず以下の順序で情報を提示する。

1. **問題の説明**（現状・リスク）
2. **正規の解決ルート**（自力でできる方法：廃止許可・地目変更登記、自主売却、解体、自治体窓口への相談など）を具体的な手順つきで先に明示する
3. **正規ルートのハードル**（時間・費用・専門知識・遠方対応の難しさなど、誇張せず客観的に）
4. **代替手段としての自社サービス**（「困難な場合の選択肢の一つ」という位置づけで、正規ルートの後に紹介する）

→ 自社サービスを記事の「解決策」として最初に出さない。必ず正規ルートの次に置く。

---

### 3. 事実確認ルール

- 法制度・手続きに関する記述（相続土地国庫帰属制度、墓地埋葬法、農地法、廃止許可、地目変更登記など）は、**執筆前に必ず一次情報（法務省・厚労省・自治体の公式ページ等）または複数の専門家サイトで裏取りする**。
- 「却下される」「認められない」等の法的結論を書く場合、根拠となる条文・ガイドラインの区分名を（読者向けにかみ砕いた形で）示す。
- 費用・期間などの数値は、実例の幅を示し、「必ず〜円かかる」という断定はしない（例：「数十万〜数百万円程度になるケースが多い」）。
- 法律・制度は改正されうるため、記事内に施行日や制度名の正式名称を明記し、最新情報の確認を読者に促す一文を入れる。

---

### 4. 有料サービス紹介・CTAのルール

- サービス紹介は「代替案」であることが伝わる文言にする（例：「〜が難しい場合の選択肢として」「〜という方法もあります」）。
- 「今すぐ」「今しかない」等の緊急性を煽るCTAは使わない。
- 無料相談・無料査定へ誘導すること自体は問題ないが、その前に「まず自治体窓口や司法書士・行政書士・宅地建物取引業者などの専門家に相談することを推奨する」旨を必ず記載する。
- 契約条件・料金体系について曖昧にせず、「一定の処分費用がかかる」等の抽象表現だけで終わらせず、目安や査定制であることを明記する。

---

### 5. トーン・語彙のガイドライン

- 断定形（「〜できません」）よりも、条件つき表現（「〜の場合は困難です」「〜すれば可能になります」）を優先する。
- 読者を「無知」扱いしない（「多くの人が誤解していますが」等は事実訂正の範囲に留め、見下すニュアンスを避ける）。
- 専門用語（地目、改葬、廃止許可、地目変更登記など）は初出時に簡潔に説明する。

---

### 6. 公開前チェックリスト

- [ ] 「絶対」「100%」「唯一」等の断定語が残っていないか
- [ ] 正規の解決ルート（自力での手続き方法）が具体的に説明されているか
- [ ] 自社サービスの紹介より前に、自治体・専門家への相談を勧める記述があるか
- [ ] 法的な記述に一次情報の裏取りが取れているか
- [ ] 数値（費用・期間）が断定でなく幅を持った表現になっているか
- [ ] CTAが緊急性を煽っていないか
- [ ] 見出しだけを読んでも「他に方法がない」と誤解されない構成になっているか
